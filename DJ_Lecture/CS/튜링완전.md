### '프로그래밍 언어' 와 ‘튜링 완전성’

현대의 모든 프로그래밍 언어는 ‘튜링 완전’ 하다. 

하지만, ‘튜링 완전성’ 이 프로그래밍 언어의 조건은 아니다. 

즉, 튜링 완전성이 없어도, 프로그래밍 언어라고 부를 수 있다는 말 이다. 

그렇다면, 프로그래밍 언어란 무엇인가? 

프로그래밍 언어를 `광의` 의 프로그래밍 언어로 보느냐, `협의` 의 프로그래밍 언어로 보느냐에 따라 달라진다. 

### 광의의 프로그래밍 언어 (# 넓은 범주의 프로그래밍 언어)

```jsx
'컴퓨터에게 명령' 을 할 수 있으면, '프로그래밍 언어' 라고 보는 입장.

정리하면, 
프로그래밍 언어 = 컴퓨터에 명령을 표현하는 형식 언어
→ SQL, Regex, Coq, HTML, CSS 등 포함
→ 학계에서 일반적으로 받아들여짐
```

- 튜링완전성 : (출처 : https://www.cs.odu.edu/)
    
    해당 페이지를 읽어보겠습니다.## 튜링 완전성의 필수 조건 (ODU 페이지 요약)
    
    ### 1단계: 핵심 정의
    
    프로그래밍 언어가 임의의 튜링 머신을 시뮬레이션할 수 있다면 튜링 완전 또는 계산적으로 범용적이라고 함
    
    ---
    
    ### 2단계: 필수 조건 3가지 (Structured Program Theorem)
    
    **Boehm-Jacopini 정리**
    
    구조적 프로그램 정리는 임의로 복잡한 제어 흐름을 사용하여 계산할 수 있는 모든 것을 세 가지 특정 방식으로 코드 블록을 결합하여 계산할 수 있음을 증명:
    
    ### 필수 3요소
    
    ```
    1. 순차 실행 (Sequence)
       → 한 코드 블록을 실행한 후 다른 블록을 실행
    
    2. 선택 (Selection)
       → boolean 표현식 값에 따라 두 코드 블록 중 하나 선택
       → if-then-else 구조
    
    3. 무한 반복 (Unbounded Iteration)
       → boolean 표현식이 참이 될 때까지 코드 블록 반복
       → while 루프
    
    ```
    
    ---
    
    ### 3단계: 실용적 정리
    
    실용적으로, 프로그래밍 언어가 직선 코드 시퀀스, if-then-else 형태, 그리고 무한 반복(예: while 루프) 형태를 허용한다면 튜링 완전
    
    ---
    
    ### 4단계: 대안 - 재귀로 대체 가능
    
    반복을 통해 수행할 수 있는 모든 것은 재귀를 통해서도 수행 가능하며 그 반대도 마찬가지
    
    **대안 조건:**
    
    ```
    1. 순차 실행 (Sequence)
    2. 선택 (Selection)
    3. 무한 재귀 (Unbounded Recursion)
    
    ```
    
    ---
    
    ### 5단계: 중요한 통찰
    
    ### 산술 연산은 필수가 아님
    
    강력한 산술이나 논리 연산이 ALU에 내장될 필요는 없으며, 튜링 머신은 그런 것이 없고 단순히 심볼 매칭과 저장에만 의존
    
    ### 무한 루프 능력이 핵심
    
    튜링 머신은 확실히 무한 루프에 빠질 수 있으므로, 무한 루프에 빠질 수 없는 프로그래밍 언어는 튜링 완전할 수 없음
    
    ---
    
    ### 6단계: 실제 사례 - PDF (의도적으로 튜링 불완전하게 만든 경우)
    
    Adobe는 Postscript에서 PDF로 전환하면서 재귀와 무한 루프를 제거하여 의도적으로 튜링 완전성을 희생하고 예측 가능성을 선택
    
    **PDF가 튜링 불완전한 이유:**
    
    - 프로시저를 이름에 바인딩하여 함수로 호출 불가 → **재귀 불가능**
    - 무한 루프 연산자(`loop`) 완전 제거 → **무한 반복 불가능**
    
    ---
    
    ### 7단계: 요약표
    
    | 필수 조건 | 설명 | 대안 |
    | --- | --- | --- |
    | **1. 순차 실행** | 코드를 순서대로 실행 | - |
    | **2. 조건 분기** | if-then-else | - |
    | **3. 무한 반복** | while 루프 | **OR** 무한 재귀 |
    
    ---
    
    ### 최종 정리
    
    **튜링 완전성의 절대 필수 조건:**
    
    ```
    ✅ 순차 실행 (Sequence)
    ✅ 조건 선택 (Selection / if-then-else)
    ✅ 무한 반복 (Unbounded Iteration)
       OR
       무한 재귀 (Unbounded Recursion)
    
    ❌ 사칙연산 (필수 아님 - 위 3가지로 구현 가능)
    ❌ 복잡한 ALU 연산 (필수 아님)
    
    ```
    
    **핵심 인사이트:**
    
    - 제어 로직이 중요 (산술 능력은 부차적)
    - 무한 루프 또는 재귀 중 하나는 **반드시** 필요
    - 예측 가능성을 위해 의도적으로 튜링 완전성을 제거할 수도 있음 (PDF 사례)

### 튜링 완전성 이란?

```jsx
1. 순차 실행 (Sequence)
   → 한 코드 '블록'을 실행한 후 다른 '블록'을 실행

2. 선택 (Selection)  
   → boolean 표현식 값에 따라 '두 코드 블록' 중 '하나' 선택
   → if-then-else 구조

3. 무한 반복 (Unbounded Iteration) OR '재귀'
   → boolean 표현식이 참이 될 때까지 코드 블록 반복
   → while 루프
   -> 대안 : 재귀
```

### 굳이, 반복(재귀) 함수(계산) 까지 필요하지 않아, `튜링 완전성` 을 포기하는 경우

```jsx
튜링 완전성의 핵심요소는 '반복(재귀, 무한루프) 능력' 다. 

이 무한 루프를 통해, 다양한 계산을 할 수 있다. 

하지만, 그로 인해, 발생하는 위험도 있다. '무한 루프에 빠질 수 있다.' 는 위험성이다. 

굳이, 무한루프에 빠질 위험을 감수할 만큼, 반복(재귀) 함수(계산)을 활용하지 않는 경우에는
의도적으로 튜링 완전성을 포기 하기도 한다. 

즉, 
	1) 무한 루프로 만들어낼 수 있는 함수(계산) 을 포기하는 대신 
	2) 안정성 (이 프로그램이 무한루프에 빠져서 멈추는 일은 없을거야) 을 택하기도 한다. 
	
	그 예시가 pdf 이다. 
```

### 튜링완전하면?

```jsx
튜링 완전하면, 튜링 시뮬레이션을 통해 계산할 수 있는 모든 계산을 할 수 있다? 
튜링 완전하면, '계산' 이 가능한 언어다? 

튜링 완전하면, 예측 가성성이 사라진다? 
trade off 관계에 있나? 
```

### 튜링 완전성 예시 코드

```jsx
순차 실행 : 한 코드 블록 실행 -> 다른 블록 실행 (#블록 단위로 실행, 됨을 의미) 

```

- [1] while 루프
    
    ```jsx
    // 1부터 N까지의 합 구하기
    function sumToN(n) {
        // 1. 순차 실행 (Sequence)
        let sum = 0;
        let i = 1;
        
        // 3. 무한 반복 (Unbounded Iteration)
        while (i <= n) {
            // 2. 조건 선택 (Selection)
            if (i % 2 === 0) {
                sum += i;  // 짝수만 더하기
            } else {
                sum += i * 2;  // 홀수는 2배로 더하기
            }
            
            i++;  // 순차 실행
        }
        
        return sum;  // 순차 실행
    }
    
    console.log(sumToN(5)); 
    // 1*2 + 2 + 3*2 + 4 + 5*2 = 2 + 2 + 6 + 4 + 10 = 24
    ```
    
- [2] 재귀 실행 (#⭐ 재귀의 의미에 대해서)
    
    ```jsx
    // 반복문 없이 재귀만 사용
    function sumToN(n) {
        // 2. 조건 선택 (Selection) - Base case
        if (n <= 0) {
            return 0;  // 종료 조건
        }
        
        // 1. 순차 실행 (Sequence)
        let current = (n % 2 === 0) ? n : n * 2;
        
        // 3. 무한 재귀 (Unbounded Recursion)
        return current + sumToN(n - 1);
    }
    
    console.log(sumToN(5));
    // 5*2 + 4 + 3*2 + 2 + 1*2 = 10 + 4 + 6 + 2 + 2 = 24
    ```
    

### 결론적으로

- 프로그래밍 언어를 광의로 보느냐, 협의로 보느냐에 따라서 HTML, CSS 를 프로그래밍 언어로 보느냐가 달라진다.
- 광의로 본다면, `컴퓨터에게 명령` 을 한다면, 프로그래밍 언어! 라고 볼 수 있다.
- 협의로 본다면, `범용 계산` 이 가능해야, 프로그래밍 언어라고 볼 수 있는데
    - 범용 계산이란, ‘상상할 수 있는 모든 알고리즘을 계산할 수 있는 상태’ 라고 볼 수 있다.
    - 범용계산이 되려면
        - 순차실행, 조건, 반복 실행이 가능해야 한다.
        - 이렇게 범용 계산이 되는 상태를 튜링 완전하다, 라고 표현하기도 한다.
    - 따라서, 일부만 계산이 가능한 SQL, HTML, CSS 는 프로그래밍 언어가 아니라고 보는 관점이다.
- 튜링 완전성을 포기하는 이유로
    - 튜링 완전성의 핵심은 `무한 루프` 인데
    - 굳이 무한 루프의 위험성을 감수할 만큼, 반복문 기능을 활용하지 않는 경우에는
    - 튜링 완전성을 포기하고, `예측 가능성(안정성)` 을 택하기도 한다.
    - 예를 들어 PDF 가 그렇다.
    - 결국, 해당 컴퓨터 언어가, `어떤 역할` 을 맡고 있느냐에 따라, 튜링 완전성(범용 계산성) 이 필요한지를 판단하게 되고, 그에 따라 해당 언어의 역할을 넣고 빼고 하게 되는 것 이라고 보면 된다.
