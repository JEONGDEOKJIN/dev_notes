


### 공부의 KPI 

```bash
- 공부의 KPI 는 '모르는 것' 을 '아는 것' 으로 '바꾸는 것' 이다. 
    - 따라서, 
        1) 전체 공부 대상 중에서, '핵심' 이 뭔지를 파악하고,
        2) 해당 주제에서 '모르는 것' 이 뭔지를 파악하고 
        3) 그걸 '알아가는 것' 이 필요하다.  
```

<br />

### 공부의 흐름 

```bash

1. '공부 타겟' 을 '리스트' 화 한다. 

2. '중요도' 를 파악한다. 
    ex) 기출 빈도 높으면 -> '핵심' 
    ex) 선생님이 중요하다고 하면 -> '핵심' 
    ex) 중요한 뉘앙스가 있으면 -> '핵심' 
    ex) 그렇지 않으면 -> '변두리' 

3. '핵심 항목' 에 대해서 '이해 암기' 를 시작한다. 
    3.1 '이해 암기' 는 '이미지' 또는 '어원' 을 활용한다. 
        - 구체적인 방법은 https://bit.ly/4eMOthN 를 확인 
        - 1~2 회독 안에 깊게 판다.

4. '변두리' 에 대해서는
    - 너무 오래 머물지는 않는다. 
    - 다음 회독때 해결한다. 
    - '단순 암기' 를 한다. 
        - 0.5 초 머무르고 -> '회상' 한다. 
        - 만약, 안 외워지면 '강도' 보다 '횟수' 를 늘린다. 

5. '이해 암기' 또는 '단순 암기' 를 했으면 -> '링크' 를 한다. 
    - 링크에 대한 구체적인 내용은 'https://bit.ly/4eMOthN' 를 확인한다. 

6. '반복' 하여 장기기억으로 만든다. 
    - 이때, '누군가에게 설명' 을 한다. 
```
