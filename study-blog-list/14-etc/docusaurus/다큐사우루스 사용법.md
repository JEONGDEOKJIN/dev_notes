# 포스트 작성하고 '메타 데이터' 추가하기

```yaml
---
# ============ 사이드바 관련 ============
sidebar_position: 3      # 사이드바에서 문서가 표시되는 순서 (숫자가 작을수록 위에 표시)
sidebar_label: '안녕!'    # 사이드바에 표시될 짧은 제목 (title이 길 때 유용)

# ============ URL/식별자 ============
slug: greetings          # 문서의 URL 경로를 커스터마이징 (예: /blog/greetings)
id: my-doc-id            # 문서의 고유 식별자 (다른 문서에서 참조할 때 사용)

# ============ 제목/설명 ============
title: 안녕하세요. 저는 DJ 입니다.!  # 문서 페이지의 메인 제목
description: 이 포스트는...         # SEO용 메타 설명 (검색 결과에 표시됨)
keywords: [docusaurus, 블로그]      # SEO용 키워드 메타 태그

# ============ 작성자 정보 (블로그용) ============
authors:
  - name: Joel Marcey              # 작성자 이름
    title: Co-creator of Docusaurus 1  # 작성자 직함/소개
    url: https://github.com/JoelMarcey  # 작성자 프로필 링크
    image_url: https://github.com/JoelMarcey.png  # 작성자 프로필 이미지
  - name: Sébastien Lorber
    title: Docusaurus maintainer
    url: https://sebastienlorber.com
    image_url: https://github.com/slorber.png

# ============ 분류/태그 ============
tags: [greetings]        # 포스트 태그 (필터링/분류에 사용)

# ============ 표시/숨김 옵션 ============
draft: false             # true면 개발환경에서만 표시 (프로덕션 빌드에서 제외)
unlisted: false          # true면 사이드바/검색에서 숨김 (직접 URL로는 접근 가능)
hide_title: false        # true면 문서 상단의 제목 숨김
hide_table_of_contents: false  # true면 우측 목차(TOC) 숨김

# ============ 네비게이션 ============
pagination_next: next-doc-id    # "다음" 버튼이 가리킬 문서 ID
pagination_prev: prev-doc-id    # "이전" 버튼이 가리킬 문서 ID
pagination_label: 다음 글 제목   # 네비게이션 버튼에 표시될 텍스트

# ============ 목차(TOC) 설정 ============
toc_min_heading_level: 2   # 목차에 포함할 최소 제목 레벨 (h2부터)
toc_max_heading_level: 3   # 목차에 포함할 최대 제목 레벨 (h3까지)

# ============ 기타 ============
image: /img/thumbnail.png  # 소셜 미디어 공유 시 표시될 썸네일 이미지
date: 2024-01-15           # 포스트 작성일 (블로그용, 파일명으로도 지정 가능)
custom_edit_url: https://github.com/...  # "이 페이지 수정" 링크 커스터마이징
---

Congratulations, you have made your first post!

Feel free to play around and edit this post as much as you like.
```

# 추가로 알아야 할 것들

## 1. Docs vs Blog 차이점
| 구분 | Docs | Blog |
|------|------|------|
| 용도 | 문서/가이드 | 시간순 포스트 |
| 사이드바 | 자동/수동 생성 | 없음 (태그로 분류) |
| 작성자 | 보통 미표시 | authors 필드 사용 |
| 날짜 | 선택적 | 필수 (파일명 또는 date 필드) |

## 2. authors.yml로 작성자 관리하기
반복되는 작성자 정보는 `blog/authors.yml`에 미리 정의:
```yaml
# blog/authors.yml
dj:
  name: DJ
  title: Developer
  url: https://github.com/dj
  image_url: https://github.com/dj.png
```
그 후 포스트에서 간단히 참조:
```yaml
authors: [dj]  # 또는 authors: dj
```

## 3. 파일명으로 날짜 지정하기
블로그 파일명에 날짜 포함 시 자동 인식:
```
2024-01-15-my-post.md
2024/01/15/my-post.md
```

## 4. 문서 ID 참조하기
다른 문서 링크할 때:
```markdown
[다른 문서 보기](/docs/다른문서id)
```

## 5. 태그 페이지
태그 설정 시 자동으로 `/blog/tags/태그명` 페이지 생성됨

## 6. last_update 필드
문서 마지막 수정 정보 표시:
```yaml
last_update:
  date: 2024-01-20
  author: DJ
```

## 7. 사이드바 카테고리 설정
폴더에 `_category_.json` 파일 생성:
```json
{
  "label": "튜토리얼",
  "position": 2,
  "collapsible": true,
  "collapsed": false
}
```
