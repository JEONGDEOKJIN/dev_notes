

# 기존의 생각 
```
- 각 테이블 마다의 모달은, 각자 자식 컴포넌트에서 관리되면 될 줄 알았다. 

- 그런데, 특정 모달은 
    - 'dim' 처리가 없고, 
    - 클릭된 cell 의 바로 밑에 붙어야 한다는 요구 사항이 있었다. 
    - 그리고, table 구현상, 모든 row 마다 각자의 onClick 이 붙어 있고, 각 row 마다 모달 상태를 관리하면, -> 복수의 모달이 뜨게 되는 문제가 발생했다. 

    - 원인은 각 row 자식 컴포넌트 마다, 각자의 상태를 갖고 있었기 때문. 

    - 당연히, 부모 컴포넌트에서 모달 상태를 관리하면, 하나의 상태로 모달 처리가 가능하다. (#지금 생각해보면, 당연한건데, 상태 관리 측면에서, 설계가 제대로 안 되었었다. #그리고, 코드가 아니라, 설계 레벨에서 문제가 있으면, 그 원인을 파악하기가 힘들고, 시간을 많이 쓰게 되는 것 같다. ) 

```
