새로운 페이지를 만들 때 
‘이거랑 비슷한 컴포넌트 어디에 있었지?’ 라는 생각을 하게 됨 (#📛 이게 맞을지는 모르겠어) 

1.1 ‘기존에 만들었던, 비슷한 컴포넌트를 찾아보자!’ 라는 생각을 하게 됨 
1.2 가져오는 방법 
화면 기획서를 보고 -> 해당 페이지로 가서 -> 가져온다. 
샵바이 방식처럼, 컴포넌트가 있는 페이지로 가서 가져올 수는 없나 (#📛📛📛) 

1.3 주의해야 할 것 
내가 쓰는 코드에 ‘익숙’ 하게 된다. 내가 정한 props, 내가 정한 directory 등등 
‘익숙’ 하면, ‘빠르게’ 찍어낼 수 있다. 
하지만, 더 빠른 방법이 있을 수 있다. (#생산성의 측면에서, 더 나아질 수 있는 부분이 분명 있을 것 이다.) 
‘익숙’ 해서 ‘생산성이 올라가는 것’ 에 만족해서는 안 된다. 
‘효율적인 구조의 개선’ 을 통해 ‘생산성이 올라가는 것’ 을 추구해야 한다. 
왜냐면, 자주 사용하면, 무엇이든 익숙해진다. 그것이 좋은 도구이건, 안 좋은 도구 이건 
좋은 도구에 읷숙해질 것 인가, 안 좋은 도구에 익숙해질 것 인가.
좋은 도구로 강건한 코드를 짜서, 프로젝트 전체의 생산성을 가장 높여서, 연극을 하는 길
GridRow 컴포넌트를 사용하면서 편하다고 느꼈는데, 그게, ‘익숙함’ 인지, ‘코드 개선을 통해 생산성이 빨라지는 느낌’ 과 구별할 필요가 있다.
