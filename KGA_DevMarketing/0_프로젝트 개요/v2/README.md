# 251209 프로젝트 개요 v2 - 변경점 요약

## 변경 목적

기존 문서의 **목표(MUST-DO / NICE-TO-DO)** 와 **Project-Action-Plan** 사이의 갭을 메우기 위한 개선

---

## 주요 변경점

### 1. Phase 구분 추가

| 구분    | 내용                                        |
| ------- | ------------------------------------------- |
| Phase 1 | MUST-DO 안정화 (기존 업무 습득 + 독립 수행) |
| Phase 2 | NICE-TO-DO 착수 (자동화, Phase 1 완료 후)   |

**이유**: 기존 문서에서 MUST-DO와 NICE-TO-DO가 섞여 있어 우선순위 혼란 발생 가능

---

### 2. 기존 업무 파악 - 학습 마일스톤 추가

```
Week 1: 관찰 및 파악
Week 2: 참여 및 실습
Week 3: 독립 수행
```

**이유**: 목표에 "기존 업무를 익히고"가 있지만, 언제까지/어떤 수준으로 익혀야 하는지 기준 부재

---

### 3. 개발 vs 기존 툴 의사결정 기준 추가

```
체크리스트:
1. 기존 툴로 해결 가능한가?
2. 반복 빈도는? (주 1회 이상이면 자동화 검토)
3. 시간 비교 (기존 툴 vs 개발)
4. 오류 가능성
```

**이유**: 목표에 "개발 공수 대비 기존 툴의 효율이 좋으면 그걸 안내"가 있지만, 판단 기준 부재

---

### 4. AB TEST 범위 정의 추가

| 유형               | 개발 필요 여부 | 비고                            |
| ------------------ | -------------- | ------------------------------- |
| 광고 소재 AB TEST  | X              | 메타/구글 플랫폼 자체 기능      |
| 상세페이지 AB TEST | O (검토 필요)  | 기존 툴(VWO 등) 먼저 검토       |
| 결과 분석 자동화   | O (검토 필요)  | 스프레드시트 함수로 가능할 수도 |

**이유**: "AB TEST 자동화"가 여러 곳에 언급되지만 범위가 불명확

---

### 5. 보고서 자동화 task - 의사결정 단계 추가

```
기존: 양식 확인 → 데이터 원천 확인 → API 흐름 확인
추가: → 개발 vs Looker Studio 비교 (먼저 검토)
```

**이유**: 바로 개발로 뛰어들기 전에 기존 툴 검토 단계 필요
