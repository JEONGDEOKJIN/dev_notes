# 프로젝트 개요

# 목표
- '개발업무' + '마케팅' 업무를 '잘' 하고, '교육생 모집' 이 '잘' 되게 한다. 
- 업무를 잘 한다는 것은 'MUST-DO(기본 요구)' 를 충족하고, 'NICE-TO-DO(추가요구)' 까지 완료하는 것

- '개발업무' 의 MUST-DO는 'QA 업무' 를 빠르게 쳐내는 것.
- '개발업무' 의 NICE-TODO는 1) 기존 TOOL 이 아닌, 2) '개발이 필요할 수 밖에 없는 부분' 을 포착해서, 나의 리소스를 쓰고, 결과물로 내어내는 것 (#만약, 개발 공수 대비 기존 툴의 효율이 좋으면 그걸 안내 해야 함)

- '마케팅 업무' 의 MUST-DO 는 '기존 업무' 를 익히고 -> '광고 매체 집행' 을 '빠르게 쳐내는 것' 
- '마케팅 업무' 의 NICE-TODO 는 'AB-TEST 자동화', '광고 리포트 자동화' 해서 시간을 줄이는 것 

- '속도' 에 타협하지 않기 (#⭐⭐⭐)


# 역할 (R&R)

## 요약

```
[MUST-DO]
마케터의 역할이 우선순위 1위. (MUST-DO)
퍼블리싱 역할도 우선순위 1위. (MUST-DO)

[NICE-TO-DO]
이때, 마케터가 만나게 되는 문제에서 '기술적' 으로 해결할 수 있는 포인트를 '개발자' 로써 해결하고 싶은게 개인적인 바람이고, 이 기술이 조직 차원에 도움이 된다면 좋겠음. (#그러면 서로 윈윈!) (#NICE-TO-DO)

그렇다고, '오버 엔지니어링' 을 하고 싶은 마음은 없음.
```

## 1. 명시적으로 맡게 될 역할

### 퍼포먼스 마케팅

- 고객 구매 여정에 따른 매체 광고 기획 -> 집행 -> AB TEST -> 미디어 믹스

- 고민할 부분
  - '어떻게 하면 이 과정을 빠르게' 진행할 것 인가. (#속도 #불필요한 시간 제거 #보고서 작성하는 시간 등을 제거 #엑셀을 배워야 할지는 고민)
  - '어떻게 하면 효과적'으로 진행할 것 인가. (#정확성 #어떻게 하면 성과를 올릴 수 있을 것 인가의 문제)

### 퍼블리싱 수정 보완

- 교육 웹사이트 상세페이지 수정 사항이 들어오면 수정하기

- 고민할 부분
  - 어떻게 하면, '속도' 를 빠르게 해서, 개발자로써의 추가 업무를 할 수 있게 할 것 인가! (#속도 #'흐름을 설명할 수 있는 상태' 로 만들기 #'검색가능한상태로 만들기')

## 2. 가능성이 있는 역할

### 마케팅 전략 (#마케터 관점)

```
1) 고객 구매 여정 별 마케팅 예산
    - 광고비 지출을 할 때, 몇 퍼센트를 유입에 사용하고, 몇 퍼센트를 리타겟팅에 사용할지 등에 대한 '전략' (#왜냐면, 현재 내가 알고 있는 부분이 OLD 해서, 변경이 필요할 수 있음. #구글에서 이야기 하는 고객 구매 여정 퍼널도 8자형으로 바뀌었잖아.)

```

### 마케팅 프로세스 자동화(#초개인화 #개발자 관점)

```
1) 마케팅 보고서 자동화
    - 구글, 네이버, 카카오, 메타 등 다양한 매체 광고에 대해서 집행하게 될 것 임
    - 현재는, 부원장님이 3시간 정도의 시간을 써서 -> 리포트를 만들고 있음 (#엑셀에 이미 함수화 된 것이 있으니, 이걸 받고, 활용할 수 있는 방향도 고려)

2) AB TEST 자동화
    - 생성형 AI 사용

    - 마케팅 기획 -> 집행 -> 성과 -> 생성형 AI 에 다시 반영 -> 그에 따라서 AB TEST 를 빠르게 돌릴 수 있는 환경 구축

3) 초개인화
    - 광고 성과를 올리기 위한 '맞춤 데이터' 만들기
    - '이 맞춤 데이터' 가 과연, 성과를 제대로 이끌어 내는 가? 에 대한 고민과 생각이 필요함.

```
