## 6. AI 검색의 부상 (SGE/GEO)

### 생성형 AI 검색이란?

구글 **SGE(Search Generative Experience)**, 네이버 **Cue** 등 AI가 검색 결과를 직접 요약하고 답변을 생성하는 방식

```
기존 검색                          AI 검색 (SGE)
─────────────────────────────────────────────────────
키워드 입력 → 링크 목록 제공       질문 입력 → AI가 직접 답변 생성
사용자가 클릭해서 정보 확인        답변 내 출처 링크 포함
                                  → "제로클릭" 현상 증가
```

### 검색 시장 점유율 변화 (2025년)

| 플랫폼 | 점유율 | 비고 |
|--------|--------|------|
| 구글 | 55.5% | 10년 전 대비 크게 상승 |
| 네이버 | 30.0% | 10년 전 80%에서 하락 |
| 기타 | 14.5% | 다음, 빙 등 |

### 광고에 미치는 영향

#### 변화 1: 검색광고 노출 방식
```
기존: 검색결과 상단에 광고 노출
SGE: AI 답변 아래/옆에 "스폰서" 라벨과 함께 노출
     → 더 시각적이고, 더 설명적인 광고 문구 필요
```

#### 변화 2: 웹사이트 트래픽 감소 (제로클릭)
```
사례: '뇌경색 예방' 키워드 구글 1위 달성
      → 예상 클릭: 월 260회
      → 실제 클릭: 월 14회 (94% 감소)

원인: AI가 직접 답변을 제공 → 사용자가 웹사이트 방문 불필요
```

#### 변화 3: SEO → GEO로 진화
| 구분 | SEO (기존) | GEO (신규) |
|------|-----------|-----------|
| 목표 | 검색 결과 상위 노출 | AI 답변의 출처로 인용되기 |
| 전략 | 키워드 최적화, 백링크 | 신뢰성 있는 콘텐츠, 구조화된 데이터 |
| 지표 | 순위, 클릭률 | AI 인용 빈도, 브랜드 언급 |

### 마케터 대응 전략

1. **광고 문구 변화**: 대화형, 설명적 문구로 전환
2. **브랜드 콘텐츠 강화**: AI가 인용할 만한 신뢰성 있는 콘텐츠 제작
3. **구조화된 데이터**: Schema.org 마크업으로 AI 이해도 향상
4. **다채널 분산**: 검색 의존도 낮추고 소셜/영상 채널 강화

### 🔑 AI 출처로 인용되는 방법 (GEO 전략)

#### 왜 중요한가?
```
• 클릭 없이 해결되는 검색: 2024년 56% → 2025년 69%
• 가트너 예측: 2026년까지 유기적 검색 트래픽 50% 이상 감소
• 프린스턴대+조지아공대 연구: GEO 적용 시 AI 답변 내 가시성 30-40% 향상
```

#### GEO 핵심 전략 5가지

**1. 명확한 정의 문장으로 시작**
```
❌ 나쁜 예:
"오늘은 GEO에 대해 이야기해보려고 합니다.
 요즘 AI 검색이 많아지면서..."

✅ 좋은 예:
"GEO(Generative Engine Optimization)는 생성형 AI가 답변을
 만들 때 우리 콘텐츠를 출처로 선택하도록 최적화하는 전략이다."

→ AI는 "~는 ~이다" 형태의 정의 문장을 그대로 인용하기 쉬움
```

**2. 질문-답변 구조로 콘텐츠 설계**
```
사용자 질문 예측 → 제목/소제목에 질문 반영 → 바로 아래 명확한 답변

예시:
Q: "2024년 메타 광고 전환율은?"
A: "2024년 메타 광고의 평균 구매 전환율은 50.8%로 집계되었다."
```

**3. E-E-A-T 신뢰도 강화**
| 요소 | 의미 | 적용 방법 |
|------|------|----------|
| **Experience** | 경험 | 실제 사례, 데이터 기반 인사이트 |
| **Expertise** | 전문성 | 저자 소개, 자격/경력 표시 |
| **Authoritativeness** | 권위 | 공식 데이터, 논문, 통계 인용 |
| **Trustworthiness** | 신뢰 | 출처 명시, 투명한 회사 정보 |

**4. 구조화된 데이터 (Schema Markup)**
```html
<!-- FAQ Schema 예시 -->
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [{
    "@type": "Question",
    "name": "퍼스트파티 데이터란?",
    "acceptedAnswer": {
      "@type": "Answer",
      "text": "퍼스트파티 데이터는 기업이 자사 채널에서
              직접 수집한 고객 데이터를 말한다."
    }
  }]
}
</script>

→ AI가 콘텐츠 구조를 더 잘 이해하고 인용하기 쉬워짐
```

**5. 권위 있는 출처 인용**
```
• 정부/공공기관 데이터: 통계청, KOBACO, 과학기술정보통신부
• 업계 리포트: 나스미디어, 메조미디어, DMC리포트
• 글로벌 기관: Gartner, eMarketer, WARC
• 학술 논문: 구글 스칼라, 학회지

→ 신뢰할 수 있는 출처를 인용한 콘텐츠를 AI도 신뢰함
```

## 7. 초개인화와 퍼스트파티 데이터

### 쿠키리스 시대 배경

```
┌─────────────────────────────────────────────────────────┐
│  서드파티 쿠키 (3rd Party Cookie)                        │
│  - 다른 웹사이트 방문 기록 추적                           │
│  - 리타겟팅 광고의 핵심                                  │
│  - 개인정보 보호 이슈로 제한 추세                         │
│                                                          │
│  2024: 구글 크롬 1% 사용자 대상 쿠키 제한 실험            │
│  2025: 구글 전면 철회 발표 → 그러나 추세는 유지           │
│        (애플 Safari, Firefox는 이미 차단 중)             │
└─────────────────────────────────────────────────────────┘
```

### 데이터 유형 비교

| 구분 | 수집 주체 | 예시 | 특징 |
|------|----------|------|------|
| **퍼스트파티** | 자사 직접 수집 | 웹사이트 행동, 구매이력, 이메일 구독 | 가장 신뢰성 높음, 정책 영향 적음 |
| **제로파티** | 고객이 자발적 제공 | 설문, 선호도 설정, 위시리스트 | 가장 정확, 수집 비용 높음 |
| **서드파티** | 외부 업체 수집 | 타사이트 방문기록, 쿠키 추적 | 규제로 점점 사용 어려워짐 |

### 퍼스트파티 데이터 활용

```
수집 가능한 데이터:
├─ 유입 경로 (UTM 파라미터)
├─ 로그인/로그아웃 정보
├─ 특정 페이지 조회 (상품 상세, 가격표 등)
├─ 장바구니 추가/삭제
├─ 구매/재구매 여부
└─ 이메일 오픈율, 클릭률
```

### 🔑 퍼스트파티 데이터 수집 방법

#### 데이터 유형 2가지

| 유형 | 설명 | 예시 |
|------|------|------|
| **엔티티 데이터** | 사용자가 "누구인지" | 이름, 이메일, 나이, 성별, 위치 |
| **이벤트 데이터** | 사용자가 "무엇을 했는지" | 클릭, 페이지 조회, 장바구니, 구매 |

#### 수집 접점 (Touch Points)

```
┌─────────────────────────────────────────────────────────────┐
│                    퍼스트파티 데이터 수집 접점                 │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  1️⃣ 회원가입/로그인                                          │
│     ├─ 이메일, 이름, 전화번호                                 │
│     ├─ 소셜 로그인 (구글, 카카오, 네이버)                      │
│     └─ 마케팅 수신 동의                                       │
│                                                              │
│  2️⃣ 웹사이트/앱 행동                                         │
│     ├─ 페이지 조회 (어떤 상품을 봤는지)                        │
│     ├─ 검색어 (무엇을 찾았는지)                               │
│     ├─ 체류 시간 (얼마나 관심 있는지)                          │
│     └─ 스크롤 깊이 (콘텐츠 소비 정도)                          │
│                                                              │
│  3️⃣ 전환 행동                                                │
│     ├─ 장바구니 추가/삭제                                     │
```

#### GA4 이벤트 설정 (실무 가이드)

**이벤트 4가지 유형**

| 유형 | 설명 | 예시 |
|------|------|------|
| **자동 수집** | GA4 설치 시 기본 수집 | page_view, session_start, first_visit |
| **향상된 측정** | 설정만 켜면 수집 | scroll, click, file_download, video_start |
| **추천 이벤트** | 구글 권장 이름 사용 | add_to_cart, purchase, sign_up |
| **맞춤 이벤트** | 직접 정의 | button_click, banner_view 등 |

**이커머스 필수 이벤트**

#### 광고 플랫폼 연동 (서버 사이드 추적)

**Meta CAPI (Conversions API)**
```
기존 픽셀 (브라우저)          CAPI (서버)
─────────────────────────────────────────
쿠키 차단 시 수집 불가        쿠키 영향 없음
iOS 14+ 제한                 제한 우회 가능
데이터 손실 20-30%           손실 최소화
```

설정 방법:
1. Meta 이벤트 관리자 → Conversions API 설정
2. 서버에서 직접 이벤트 전송 (또는 파트너 통합 사용)
3. 이벤트 매칭 품질 점수 확인 (7점 이상 권장)

**Google Enhanced Conversions**
```
기존 전환 추적               Enhanced Conversions
─────────────────────────────────────────────────
쿠키 기반                    해싱된 고객 데이터 활용
정확도 낮아짐                정확도 유지/향상
```

#### 데이터 수집 체크리스트

```
[ ] 기본 설정
    ├─ GA4 속성 생성 및 데이터 스트림 설정
    ├─ GTM 컨테이너 생성 및 웹사이트 연동
    └─ 기본 페이지뷰, 이벤트 수집 확인

[ ] 이벤트 설정
    ├─ 향상된 측정 활성화 (스크롤, 클릭, 동영상 등)
    ├─ 이커머스 이벤트 설정 (view_item, add_to_cart, purchase)
    └─ 맞춤 이벤트 정의 (CTA 클릭, 폼 제출 등)

[ ] 광고 플랫폼 연동
    ├─ Meta 픽셀 + CAPI 설정
    ├─ Google Ads 전환 추적 + Enhanced Conversions
    ├─ 네이버 전환 추적 스크립트
    └─ 카카오 픽셀 설정

[ ] 데이터 품질 관리
    ├─ 이벤트 디버깅 (GA4 DebugView, Meta 테스트 이벤트)
    ├─ 이벤트 매칭 품질 점수 확인
    └─ 주요 이벤트(전환) 지정
```

## 8. 미디어 믹스 최적화 팁

### ✅ Do (해야 할 것)
- 채널별 성과를 **UTM 파라미터**로 정확히 추적
- A/B 테스트로 소재/타겟 지속 최적화
- 성과 좋은 캠페인에 예산 **15-20%씩 증액**
- 단일 채널 의존 대신 **다채널 분산**

### ❌ Don't (하지 말아야 할 것)
- 단일 플랫폼에 예산 올인 (효율 급락 시 대응 불가)
- 성과 측정 없이 감으로 예산 배분
- 플랫폼 신기능(AI 자동화) 무시

### 미디어 믹스 다각화 효과
> 구글과 메타 외 채널로 미디어 믹스를 다각화한 마케터들은
> **ROAS가 최대 214%까지 개선**되는 효과를 확인했습니다.
