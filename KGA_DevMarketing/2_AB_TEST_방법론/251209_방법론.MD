# AB TEST 방법론

## 자주 묻는 질문

### Q1. A안, B안을 1:1 비교가 아니라, 3개 이상을 한다면 어떻게 해야 하지?

**현업에서는 크게 3가지 방식으로 처리합니다:**

#### 방식 1: 순차적 A/B TEST (가장 많이 사용)

```
1주차: A vs B → B 승리
2주차: B vs C → B 승리
3주차: B vs D → D 승리
→ 최종 승자: D
```

**장점**: 단순하고 해석이 명확함
**단점**: 시간이 오래 걸림

**현업 사용 사례**:
- Meta 광고: 2~3개 소재씩 테스트 → 승자만 남기고 새 소재 추가
- 랜딩페이지: 헤드라인 A/B → 승자 확정 → CTA 버튼 A/B → 승자 확정

---

#### 방식 2: 동시 다중 비교 (A/B/n TEST)

```
동시에: A vs B vs C vs D
→ 트래픽을 25%씩 분배
→ 일정 기간 후 전체 비교
```

**통계 처리 방법**:

| 방법 | 설명 | 사용 시점 |
|------|------|----------|
| **Bonferroni 보정** | α를 비교 횟수로 나눔 (0.05/6=0.0083) | 간단하지만 보수적 |
| **Chi-Square 독립성 검정** | 전체 그룹 간 차이 유무 확인 | 먼저 차이가 있는지 확인 |
| **Tukey HSD** | 어떤 쌍이 다른지 사후 검정 | Chi-Square 후 상세 분석 |

**현업 사용 사례**:
- Meta Advantage+: 자동으로 여러 소재 테스트
- Google Optimize: 다변량 테스트 지원

---

#### 방식 3: Multi-Armed Bandit (MAB)

```
초기: A, B, C, D에 균등 분배
→ 실시간으로 성과 좋은 쪽에 트래픽 집중
→ 자동 최적화
```

**장점**: 테스트 기간 동안에도 매출 손실 최소화
**단점**: 통계적 유의성 판단이 어려움

**현업 사용 사례**:
- Meta: 자동 예산 분배
- Google Ads: 스마트 입찰

### Q2. 이걸 제대로 하려면, 뭘 공부해야 하지?

#### 기초 (필수)

| 주제 | 내용 | 학습 시간 |
|------|------|----------|
| **기초 통계** | 평균, 표준편차, 정규분포 | 1-2일 |
| **가설 검정** | p-value, 유의수준, 귀무가설 | 2-3일 |
| **Z-검정 / t-검정** | 두 그룹 평균 비교 | 1-2일 |
| **카이제곱 검정** | 범주형 데이터 비교 (전환/미전환) | 1일 |

#### 중급 (3개 이상 비교 시)

| 주제 | 내용 | 학습 시간 |
|------|------|----------|
| **다중 비교 보정** | Bonferroni, Holm, FDR | 1-2일 |
| **ANOVA** | 3개 이상 그룹 평균 비교 | 2-3일 |
| **사후 검정** | Tukey, Scheffé, Dunnett | 1-2일 |

#### 고급 (자동화/최적화)

| 주제 | 내용 | 학습 시간 |
|------|------|----------|
| **Bayesian A/B Test** | 사전 확률 + 데이터 → 사후 확률 | 1주 |
| **Multi-Armed Bandit** | Thompson Sampling, UCB | 1주 |
| **Sequential Testing** | 조기 종료 가능한 테스트 설계 | 3-5일 |

---

### 현업에서 실제로 가장 많이 쓰는 방식

```
1. 대부분의 경우: 2개씩 순차 비교 (A/B TEST)
   → 가장 단순하고 해석이 명확

2. 시간이 없을 때: Meta/Google의 자동 최적화 기능 사용
   → Advantage+, Performance Max

3. 정확한 분석 필요 시: 동시 비교 + Bonferroni 보정
   → 마케터가 직접 할 일은 거의 없음 (도구가 해줌)
```
