- [2] events 객체를 다른 파일에서 호출해서 바인딩 한다.
    
    ```bash
    데이터를 가져올 때, 
    1) events-data 파일에서 데이터를 직접 가져옴
    2) API 호출 없음 (fetch, axios 등 없음)
    3) 서버 사이드 데이터 fetching 없음 (async 함수 아님)
    ```
    
    ![image.png](attachment:bf538ca8-3666-4e7e-8a9f-6147e05d8efb:image.png)
    
- [3]  이 방식이 `정적` 이라고 불리는 이유를 `호출` 과정부터 살펴보면
    
    
    ```bash
    순서대로 생각하면 
    
    [빌드시점]
    1) 그 전에 서버에는 
    	a) 이미 npn run build 를 통해, 빌드가 완료된 파일이 들어가 있음.  
    
    2) 생성된 빌드 파일을 서버에 배포
    
    [런타임]
    2) 사용자가 브라우저에 /event 페이지를 호출 
    3) 브라우저는 서버에 요청
    4) 서버는, 이미 데이터가 전부 바인딩 된 HTML 파일을 응답해줌
    
    [데이터 변경시]
    5) 만약, HTML 에 다른 데이터를 넣고 싶으면, 다시 빌드를 해서 서버에 넣어야 함 
    6) 다시 빌드 -> 재배포 필요
    ```
