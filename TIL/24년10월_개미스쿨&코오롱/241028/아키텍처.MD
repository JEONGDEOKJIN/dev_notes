


### 이번, 천사일지에서 적용해볼 아키텍처 (디렉토리 흐름)


```BASH

api 
	|- 네트워크 관련된 핸들링 
	|- error handling
	|- retry 요청
	|- 마이크로프론트엔드 형태(#❓)
	|- network 관련 모든 요청
	|- cache 관리

repository 
	|- api 에서 받아온 것 중에서, 필요한 거랑 , 없는게 있음. -> 필요한 것만 필터링 하는 역할) 

service 
	|- 백엔드 서버에서 받아오는거
	|- 필터링 된 데이터를 가공하는 로직, a랑 b 를 합쳐서 뭔가를 만든다. ex) 데이터를 가공

hook
	|- 이렇게 가공된 걸, store 에 넣는다. 
	|- 데이터를 직접 관리, 꺼내오는. 

view 
	|- 데이터를 UI 에 바인딩 한다. 
	|- page 
	|- blocks(페이지 끼리 공통 요소, 로직이 있는 뷰) 
	|- element(스타일만 있는 것) 

```
